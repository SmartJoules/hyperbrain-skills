The Agent Skill Suite

Use the smallest skill that matches the job. Load the full suite only for large work that needs planning, delegation, execution, and integration.

1

Agent Orchestration

Use first when the request is large, ambiguous, cross-repo, or risky. It creates the roster, dependencies, ownership boundaries, and verification gates.

  • Planning agents
  • Specialist roles
  • Execution order
2

Agent Context Manager

Use before dispatching agents. It builds small context packets so each worker gets the right files, constraints, contracts, and evidence.

  • Token budgets
  • Context packets
  • Result compression
3

Delegation Contracts

Use when writing the actual agent prompt. It defines mission, scope, non-scope, tools, permissions, acceptance criteria, output schema, and stop conditions.

  • Read-only briefs
  • Scoped write briefs
  • Retry prompts
4

Fleet Runner

Use when multiple agents should run in parallel or staged pipelines. It handles worker status, model tiering, retries, cost control, and execution modes.

  • Parallel research
  • Parallel build
  • Cross-check

Recommended Workflow

The pattern is simple: plan, packet, brief, run, integrate. Each phase narrows uncertainty before the next one spends effort.

01

Classify the request

Decide if this is a solo task, parallel research, contract-first build, cross-check, or full fleet run.

02

Create the agent roster

Assign planning, research, architecture, implementation, QA, review, and integration roles only where useful.

03

Prepare context packets

Send the minimum context each agent needs: repo instructions, relevant files, contracts, constraints, and evidence.

04

Write delegation contracts

Make each prompt crisp: mission, scope, non-scope, allowed tools, output schema, verification, and stop conditions.

05

Run the fleet

Execute independent workers in parallel, sequence dependent work, track status, and retry only with narrower prompts.

06

Integrate and verify

Reconcile contracts, inspect conflicts, run tests or QA, document residual risks, and decide ready, ready-with-risk, or not-ready.
