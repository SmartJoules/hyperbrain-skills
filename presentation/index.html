what it is

A shared brain for your coding agents

Installed into your AI coding agent. Skills load automatically by relevance, or you call them by name.

Agents that do work

Reviewers, planners, generators, analyzers, optimizers — callable workers that review code, plan features, generate APIs, optimize queries, and find root causes.

Repo knowledge bases

Precomputed context for JouleTRACK, jt-api-v2, iot-feedback-handler, Lumen and the ontology service — so the AI knows our real architecture without re-reading the codebase.

Standards and patterns

Engineering standards, design patterns, language conventions and design tokens — encoded once, applied on every change.

Works everywhere

Claude Code, Pi, Codex, Cursor and Copilot — one ./install.sh per tool. Same skills, any agent.

benefits

Why it moves the needle

What the team gets from adopting it.

Higher productivity

Reuse battle-tested workflows instead of re-deriving them. Planning, TDD, review and deploy — all on rails.

Consistency by default

Every dev and every AI agent follows the same SOLID, patterns and resilience standards and the real repo conventions — fewer review cycles.

Lower token cost

Retrieve-don't-read and decompose-don't-accumulate keep large work cheap — tokens scale with sub-tasks, not files times tasks.

Quality gates built in

Design-doc gate, code review, architecture review and secret-handling standards — issues caught before merge, in CI and in the agent.

Multi-agent ready

Orchestrate a fleet with tokensmax — parallel scoped agents, model tiers and worktree-isolated writes, then merge and verify.

Compounding context

Decisions, contracts and patterns persist into knowledge bases and memory, so each task starts smarter and reads less than the last.

how to use it

Install once, then call by name

One install per tool. Skills auto-load by relevance, or you invoke them explicitly.

# install for your agent (per tool)
git clone https://github.com/SmartJoules/hyperbrain-skills.git hyperbrain-skills && cd hyperbrain-skills
./install.sh                 # Claude Code
./install.sh --assistant pi  # Pi (pi.dev)
                              # also: --assistant codex | cursor | copilot

# use it
Claude Code : "use the code-reviewer skill on this diff"   # or it auto-loads
Pi          : /skill:design-doc-review-agent docs/design/widget-design.md
Fleet (CLI) : /tokensmax build the comfort-index widget end-to-end
auto-load by task call by name /skill:<name> in Pi /tokensmax fleet

Full reference: USAGE_GUIDE.md (which skill when · multi-agent · low-token mode) and AGENTS_AND_SKILLS.md (full catalog).

one complete SDLC flow

Add a comfort-index widget to JouleTRACK

End to end — where HyperBrain helps at each stage, and which skills do the work.

1 · Ideate and frame

Turn the request into a framed problem with goals, non-goals and a few approaches before any code.

superpowers-brainstormexpert-personas

2 · Design doc

Author from the FAANG-style template, get human approval, then the review agent and CI gate verify it — under 15 pages, required sections, repo-aligned, no hardcoded secrets — and reject with fixes if not.

design-doc-harnessdesign-doc-review-agentsoftware-architecture-planner

3 · Plan and decompose

Break into scoped sub-tasks — API, query, UI — with file lists and contracts. Pull repo context from knowledge bases cheaply, no whole-module reads.

agentic-engineeringbackend-implementation-plannergraphify-integration

4 · Build — multi-agent, repo-grounded

Dispatch scoped agents in parallel: the jt-api-v2 endpoint, the Influx query, and the Angular widget with --n-* tokens — each agent gets only its slice plus the right knowledge base.

agent-fleet-runnertokensmaxjouletrack-angularbackend-knowledge-baseapi-service-generator

5 · Optimize and harden

Tune the time-series query — downsample, cache with eviction, fewer DB calls — pick the right algorithm, and enforce resilience and connection standards.

database-query-optimizeralgorithm-pickerengineering-standards

6 · Test — TDD

Tests first, 80 percent or more coverage — unit, integration, and the critical user flow end to end.

tdd-workflowplaywright-patterns

7 · Review

Code review for quality, security and performance, plus architecture review for the changed surface, then reconcile the agents' outputs before the PR.

code-reviewerarchitecture-revieweragent-integration-reviewer

8 · Ship

Deployment plan and CI/CD, secrets from the secret store and never hardcoded, with safe rollout and rollback.

devops-deployment-plannersj-k8s-knowledge-base

9 · Operate and learn

If something breaks, root-cause from logs and traces. Persist new contracts and decisions back into the knowledge bases so the next task is cheaper.

root-cause-analyzerself-learninglocal-kb

Net effect: the orchestrator holds a small plan and structured results — not the API layer, the query engine and the dashboard module all at once. Big feature, bounded context, consistent output.
