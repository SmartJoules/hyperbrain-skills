🧠What it is

A shared library installed into your AI coding agent. Skills load automatically by relevance, or you call them by name.

🤖

Agents that do work

Reviewers, planners, generators, analyzers, optimizers — callable workers that review code, plan features, generate APIs, optimize queries, and find root causes.

📚

Repo knowledge bases

Precomputed context for JouleTRACK, jt-api-v2, iot-feedback-handler, Lumen, the ontology service & more — so the AI knows our real architecture without re-reading the codebase.

📐

Standards & patterns

Engineering standards, design patterns, language conventions, design tokens — encoded once, applied on every change.

🔌

Works everywhere

Claude Code, Pi (pi.dev), Codex, Cursor, Copilot — one ./install.sh per tool. Same skills, any agent.

✅Benefits

Why it moves the needle for the team.

🚀 Higher productivity

Reuse battle-tested workflows instead of re-deriving them. Planning, TDD, review, deploy — all on rails. Targeting a meaningful uplift in dev throughput.

🎯 Consistency by default

Every dev and every AI agent follows the same SOLID/patterns/resilience standards and the real repo conventions — fewer review cycles, fewer "not how we do it" PRs.

💸 Lower token cost

Retrieve-don't-read (graphify + KBs) and decompose-don't-accumulate keep large work cheap — tokens scale with #sub-tasks, not files × tasks.

🛡️ Quality gates built in

Design-doc gate, code review, architecture review, security & secret-handling standards — issues caught before merge, in CI and in the agent.

🧩 Multi-agent ready

Orchestrate a fleet (tokensmax) — parallel scoped agents, model tiers, worktree-isolated writes, then merge & verify. Big work, controlled cost.

♻️ Compounding context

Decisions, contracts and patterns persist into knowledge bases & memory, so each task starts smarter and reads less than the last.

⚙️How to use it

Install once, then call skills by name or let them auto-load.

# Install for your agent (per tool)
git clone https://github.com/SmartJoules/hyperbrain-skills.git /tmp/hb && cd /tmp/hb
./install.sh                 # Claude Code  → ~/.claude/skills/
./install.sh --assistant pi  # Pi (pi.dev)  → ~/.pi/agent/skills/
# also: --assistant codex | cursor | copilot

# Use it
#  Claude Code : "use the code-reviewer skill on this diff"  (or auto-loads)
#  Pi          : /skill:design-doc-review-agent docs/design/widget-design.md
#  Fleet (CLI) : /tokensmax build the comfort-index widget end-to-end

Three ways skills activate:

Auto-load — matched by the task you describe Call by name — "use <skill>" Pi slash — /skill:<name> <args> Fleet — /tokensmax <task>

Full reference: USAGE_GUIDE.md (which skill when · multi-agent · low-token mode) and AGENTS_AND_SKILLS.md (full catalog).

🔄One complete SDLC flow

Example: "Add a Comfort-Index widget to JouleTRACK" — end to end, showing where HyperBrain helps at each stage.

1

Ideate & frame

Turn the request into a framed problem with goals/non-goals and 2–3 approaches before any code.

superpowers-brainstormexpert-personas
2

Design doc

Author from the FAANG-style template; get human approval; the review agent + CI gate verify it (≤15 pages, required sections, repo-aligned, no hardcoded secrets) — reject & suggest fixes if it's not.

design-doc-harnessdesign-doc-review-agentsoftware-architecture-planner
3

Plan & decompose

Break into scoped sub-tasks (API · query · UI) with file lists & contracts. Pull repo context from KBs cheaply — no whole-module reads.

agentic-engineeringbackend-implementation-plannergraphify-integration
4

Build (multi-agent, repo-grounded)

Dispatch scoped agents in parallel: jt-api-v2 endpoint (Sails/Waterline), the Influx query, and the Angular widget (OnPush, --n-* tokens) — each agent gets only its slice + the right KB/standard.

agent-fleet-runnertokensmaxjouletrack-angularbackend-knowledge-baseapi-service-generator
5

Optimize & harden

Tune the time-series query (downsample, cache with eviction — fewer DB calls), pick the right algorithm, and enforce resilience & connection standards.

database-query-optimizeralgorithm-pickerengineering-standards
6

Test (TDD)

Tests first, 80%+ coverage — unit, integration, and the critical user flow E2E.

tdd-workflowplaywright-patterns
7

Review

Code review (quality/security/perf) + architecture review for the changed surface + integrate/reconcile the agents' outputs before the PR.

code-reviewerarchitecture-revieweragent-integration-reviewer
8

Ship

Deployment plan + CI/CD; secrets from the secret store (never hardcoded); safe rollout & rollback.

devops-deployment-plannersj-k8s-knowledge-base
9

Operate & learn

If something breaks: root-cause from logs/traces. Persist new contracts & decisions back into the KBs so the next task is cheaper.

root-cause-analyzerself-learninglocal-kb

Net effect: the orchestrator holds a small plan + structured results — not the API layer, the query engine, and the dashboard module all at once. Big feature, bounded context, consistent output.
