## 🎯 What is HyperBrain?

HyperBrain is a **comprehensive AI-powered skill library** for software development that transforms Claude Code into an expert development companion. It combines:

- **22 Technical Skills** - Domain-specific workflows and patterns
- **4 Knowledge Bases** - 48,803 words of organizational knowledge
- **7 MCP Servers** - 3 always-on + 4 on-demand integrations
- **7 Expert Personas** - Role-specific AI behavior
- **Self-Learning System** - Continuous improvement through user context

### 6. Self-Learning System

**Location:** `self-learning/SKILL.md` + `memory.js`

**Learning Pipeline:**
```
Interaction → Capture Context → Store → Analyze → Improve → Apply
```

**What It Learns:**
- Coding preferences (patterns, frameworks, conventions)
- Project context (structure, team practices, deployment)
- Successful patterns (what works well)
- User feedback (corrections, preferences)
- Domain knowledge (terminology, APIs, schemas)

**4 Personalization Levels:**
- **Level 1 (0-100 interactions):** Basic profile, languages, style
- **Level 2 (100-500):** Patterns, testing approach, documentation
- **Level 3 (500-1000):** Project structure, team practices, domain
- **Level 4 (1000+):** Workflow patterns, proactive suggestions

**Storage Location:** `~/.claude/memory/`

**CLI Commands:**
```bash
node memory.js init              # Initialize memory system
node memory.js stats             # Show learning statistics
node memory.js export [path]     # Export memory to JSON
node memory.js clear [days]      # Clear old interactions (90-day default)
node memory.js optimize          # Optimize learned patterns
```

**Privacy Features:**
- ✅ All data stored locally (never leaves machine)
- ✅ No cloud sync or sharing
- ✅ Encrypted at rest
- ✅ Automatic data retention (90 days)
- ✅ Full user control

## 🎯 UI/UX Design Rules (3 MANDATORY)

**Location:** `ui-ux-design/SKILL.md`

### 1. Single Page Flow (MANDATORY)
- Always prefer single-page dashboards over routing
- Show everything on one screen when possible
- Use Angular's component-based architecture

### 2. Mobile Responsiveness (MANDATORY)
- Mobile-first design is NOT optional
- Breakpoints: Mobile (320px+), Tablet (768px+), Desktop (1024px+)
- Touch targets: Minimum 44x44px
- 8px grid system for spacing

### 3. Design Consistency (MANDATORY)
- Consistent colors (use design tokens)
- Consistent typography (font families, sizes, weights)
- Consistent spacing (8px grid)
- Consistent component behavior

## 🚀 Installation & Usage

### Quick Install (3 Steps)
```bash
# 1. Clone the repository
git clone https://github.com/itsatif/hyperbrain-skills.git ~/.claude/skills/hyperbrain-skills

# 2. Run installation script
cd ~/.claude/skills/hyperbrain-skills
./install.sh

# 3. Verify installation
ls -la ~/.claude/skills/
```

### Manual Install
```bash
# Copy skills to Claude skills directory
cp -r ~/.claude/skills/hyperbrain-skills/* ~/.claude/skills/

# Copy MCP configs
cp -r ~/.claude/skills/hyperbrain-skills/mcp-setup/* ~/.claude/

# Configure environment
cp ~/.claude/skills/hyperbrain-skills/mcp-on-demand/.env.example ~/.claude/skills/hyperbrain-skills/mcp-on-demand/.env
```

## 🔧 Configuration

### Enable/Disable Features

**Learning System:**
```json
// ~/.claude/settings.json
{
  "learning": {
    "enabled": true,
    "storage_location": "~/.claude/memory",
    "retention_days": 90,
    "anonymize_data": true
  }
}
```

**MCP Servers:**
```json
// ~/.claude/skills/hyperbrain-skills/mcp-on-demand/mcp-config.json
{
  "mcpServers": {
    "morpheus": {
      "disabled": false,
      "autoApprove": ["mcp__morpheus__*"]
    }
  }
}
```

## 🎓 Best Practices

### When to Use HyperBrain
✅ Complex feature development
✅ Architecture decisions
✅ Code review and quality
✅ Test generation
✅ Documentation
✅ Onboarding new developers
✅ Cross-team collaboration

### When NOT to Use
❌ Simple one-line fixes
❌ Basic syntax questions
❌ Quick lookups (use documentation instead)

### Getting the Most Out of HyperBrain

1. **Always Let Superpowers Activate First**
   - Don't skip the brainstorming phase
   - Answer questions thoughtfully
   - Provide context about your project

2. **Let Expert Personas Guide You**
   - Trust the persona's expertise
   - Follow the suggested approach
   - Ask questions if you don't understand

3. **Check Knowledge Bases Before Coding**
   - Search for existing implementations
   - Reuse proven patterns
   - Don't reinvent the wheel

4. **Use On-Demand MCPs Wisely**
   - Only activate when needed
   - Use for complex queries
   - Check local knowledge first

5. **Let the Learning System Improve**
   - Provide feedback when responses are good/bad
   - Correct mistakes explicitly
   - Use consistently for best personalization
